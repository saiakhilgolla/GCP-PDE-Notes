Vertex AI Batch Prediction

1. Core Concepts

What is Vertex AI Batch Prediction?

  • Asynchronous prediction service for large datasets
  • Processes predictions in batches rather than real-time
  • Optimized for throughput over latency
  • Cost-effective for bulk inference workloads

Key Characteristics

AspectBatch PredictionOnline Prediction
LatencyMinutes to hoursSub-second
ThroughputVery highLimited by endpoint
CostLower per predictionHigher per prediction
Use CaseBulk processingReal-time serving
InfrastructureAuto-scaling workersAlways-on endpoints

Input Data Sources

  • BigQuery tables - Most common for structured data
  • Cloud Storage - JSONL, CSV, TFRecord formats
  • Direct file uploads for smaller datasets

Output Destinations

  • BigQuery table - Structured output with metadata
  • Cloud Storage bucket - JSONL format output
  • Output includes predictions + original input data

  2. Resource Allocation - Auto-scale worker nodes
  3. Data Chunking - Split input into processable batches
  4. Model Loading - Load model artifacts to workers
  5. Parallel Processing - Execute predictions across workers
  6. Result Aggregation - Collect and format outputs
  7. Output Writing - Write to destination with metadata

4. Exam Traps

Common Misconceptions

  • Trap: Thinking batch prediction is real-time → It's asynchronous
  • Trap: Using batch for low-latency needs → Use online prediction instead
  • Trap: Expecting immediate results → Jobs take minutes to hours
  • Trap: Not considering output schema → Includes input + predictions

Configuration Gotchas

IssueProblemSolution
Format mismatchInput format != model expectationMatch model's training format exactly
BigQuery schemaColumn names don't matchUse exact column names from training
Resource limitsJob fails on large datasetsCheck 100M instance limit
GPU allocationRequesting GPUs for CPU modelsOnly use GPUs for GPU-trained models
Output permissionsCan't write to destinationCheck IAM permissions on output location

Decision Framework Traps

  • Wrong: "Need predictions for 1M records" → Batch prediction
  • Correct: Consider latency requirements first
  • Wrong: "Real-time dashboard" → Batch prediction
  • Correct: Real-time = Online prediction
  • Wrong: "Process nightly" → Online prediction
  • Correct: Scheduled bulk = Batch prediction

Cost Optimization Traps

  • Trap: Over-provisioning accelerators → Use only when needed
  • Trap: Wrong machine type selection → Match model requirements
  • Trap: Not using preemptible instances → Enable for cost savings
