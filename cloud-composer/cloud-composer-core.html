Cloud Composer Study Notes

1. Core Concepts

What is Cloud Composer?

Fully managed Apache Airflow service for orchestrating complex workflows across cloud and on-premises environments. Built on Google Kubernetes Engine (GKE).

Key Components

Component Purpose Key Details
DAG (Directed Acyclic Graph) Workflow definition Python file defining tasks and dependencies
Operators Task executors BashOperator, PythonOperator, BigQueryOperator, etc.
Sensors Wait for conditions FileSensor, TimeDeltaSensor, ExternalTaskSensor
Tasks Individual work units Instance of an operator with parameters
Scheduler DAG execution engine Monitors DAGs and triggers task instances
Executor Task runner CeleryExecutor (default), KubernetesExecutor

Airflow Concepts

  • DAG Run: Execution instance of a DAG for specific date
  • Task Instance: Specific run of a task within DAG run
  • XCom: Cross-communication between tasks (key-value store)
  • Connection: External system credentials stored in Airflow
  • Variable: Global key-value pairs accessible across DAGs
  • Pool: Resource limitation mechanism for task concurrency

Feature Composer 1 Composer 2

Network Architecture

  • Private IP: GKE nodes use private IPs only
  • Authorized Networks: Control web server access
  • VPC Native: Uses alias IPs for pod networking
  • Private Service Connect: Composer 2 can use PSC endpoints
  • Firewall Rules: Automatic creation for internal communication

4. Exam Traps

Common Gotchas

  • DAG Import Time: Complex Python imports in DAG files slow scheduler
  • XCom Limitations: 48KB limit, not for large data transfer
  • Sensor Poke vs Reschedule: Poke mode blocks worker slots
  • Private IP Only: Workers cannot access internet without NAT gateway
  • Environment Updates: Can take 20-60 minutes to apply
  • IAM Propagation: Service account changes need time to propagate
  • Composer 1 Deprecation: New projects should use Composer 2

Scaling Anti-Patterns

Problem Wrong Solution Correct Approach
Slow task execution Add more nodes Optimize task code, use appropriate machine types
Scheduler lag Increase worker count Reduce DAG complexity, optimize parsing
