Cloud Composer - Exam Traps & Cheatsheet

Core Concepts
  • Managed Apache Airflow: Fully managed orchestration service built on Apache Airflow
  • DAGs (Directed Acyclic Graphs): Python files defining workflow structure and dependencies
  • Operators: Define single tasks (BashOperator, PythonOperator, BigQueryOperator)
  • Sensors: Wait for conditions or external events before proceeding
  • Environment: Kubernetes cluster running Airflow components + web server + scheduler
  • GCS Bucket: Stores DAGs, plugins, data, and logs

Exam Traps

🚨 Common Gotchas

  • DAG Deployment: Upload to gs://bucket/dags/ folder, NOT root bucket
  • Dependencies: Use requirements.txt for PyPI packages, NOT pip install in operators
  • Sensors vs Operators: Sensors WAIT for conditions, operators EXECUTE actions
  • Environment Sizing: Can't change machine type after creation, must recreate environment
  • Cross-region Access: DAG can't directly access resources in different regions without proper networking

🎯 Trick Patterns

  • "Real-time processing": Composer is for BATCH orchestration, not streaming
  • "Immediate execution": DAGs have schedule intervals, minimum is 1 minute
  • "Direct data processing": Composer orchestrates OTHER services, doesn't process data itself
  • "Multi-cloud orchestration": Can orchestrate external services but runs on GCP infrastructure
  • "Cost optimization": Always running, not serverless - consider Cloud Functions for simple scheduled tasks

Environment Sizing Guide

  • Small: Dev/test, <10 concurrent tasks
  • Medium: Production, 10-100 concurrent tasks
  • Large: High-volume, >100 concurrent tasks
  • Custom: Specific machine types for special needs

Networking Options

  • Public IP: Default, internet access for external APIs
  • Private IP: VPC-only, use Cloud NAT for outbound internet
  • Private Service Connect: Secure access to Google APIs
  • Authorized Networks: Restrict web server access by IP
