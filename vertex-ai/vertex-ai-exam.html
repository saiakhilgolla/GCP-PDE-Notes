Vertex AI - Unified ML Platform

Core Concepts

Unified ML Platform

  • Vertex AI: End-to-end ML platform combining AutoML and custom training
  • Workbench: Managed Jupyter notebooks with pre-configured ML environments
  • Model Garden: Collection of pre-trained, foundation, and open-source models
  • Feature Store: Centralized repository for ML features with versioning
  • Pipelines: MLOps workflow orchestration using Kubeflow or TFX

⚠️ Exam Traps

Common Confusions

  • AutoML vs Custom Training: AutoML = no code required, Custom = full control with containers
  • Online vs Batch Prediction: Online = real-time endpoints, Batch = scheduled jobs on large datasets
  • Model Garden vs Registry: Garden = pre-built models, Registry = your trained models
  • Feature Store Online vs Offline: Online = real-time serving, Offline = training data preparation

Training Traps

  • Hyperparameter Tuning requires Custom Training - not available with AutoML
  • AutoML automatically handles feature engineering - no manual preprocessing needed
  • Custom containers must be in Container Registry or Artifact Registry - not Docker Hub
  • Training data must be in same region as Vertex AI job - no cross-region access

Serving Traps

  • Batch Prediction requires Cloud Storage output - cannot write directly to BigQuery
  • Online endpoints charge for node uptime - even with zero traffic
  • Model deployment requires explicit machine type selection - no automatic sizing
  • Edge deployment needs TensorFlow Lite conversion - not all models supported

Feature Store Gotchas

  • Online serving requires Bigtable backend - automatic provisioning
  • Feature ingestion has eventual consistency - not immediate availability
  • Point-in-time queries only available offline - not in online store
  • Schema evolution requires versioning - cannot modify existing features

Pipeline Traps

  • Vertex Pipelines uses Kubeflow SDK v2 - not compatible with v1
  • Pipeline components must be containerized - no direct Python functions
  • Artifact lineage requires explicit metadata - not automatically tracked
  • Scheduled pipelines require Cloud Scheduler integration - not built-in cron
Quick Cheatsheet

Training Decision Tree

  • Standard ML task + No ML expertise: AutoML
  • Custom algorithms + Full control: Custom Training
  • Optimize model performance: Hyperparameter Tuning
  • Large datasets (>100GB): Custom Training with distributed strategy

Serving Decision Tree

  • Real-time + Low latency: Online Prediction Endpoints
  • Large batch + Cost optimization: Batch Prediction Jobs
  • Mobile/IoT devices: Edge Deployment (TF Lite)
  • Variable traffic: Endpoint with auto-scaling
