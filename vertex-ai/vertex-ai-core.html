Exam Traps & Gotchas

Common Confusion Points

AutoML vs Custom Training:
  • AutoML: No code, limited customization, faster to deploy
  • Custom Training: Full code control, any framework, complex architectures
  • Don't choose AutoML for specialized architectures or research models
Batch vs Online Prediction:
  • Batch: High throughput, batch processing, cost-effective for bulk inference
  • Online: Real-time, sub-100ms latency, always-on cost
  • Can't use online endpoints for bulk batch processing (cost prohibitive)

Feature Store Traps

Scenario Wrong Answer Correct Answer
Real-time feature serving Query BigQuery directly Use Feature Store online serving
Feature sharing across teams Duplicate feature pipelines Centralized Feature Store
Training/serving skew Manual feature alignment Feature Store ensures consistency

Pipeline & Experiment Traps

  • Experiments: Track runs, not deploy models. Use for comparison and artifact logging.
  • Pipelines: Orchestrate workflows, not one-time training jobs
  • Metadata: Automatic tracking of lineage, not manual logging
  • Model Registry: Centralized storage, not deployment
Monitoring Trap: Model Monitoring is for ML-specific metrics (drift, bias), not infrastructure monitoring (CPU, memory). Use Cloud Monitoring for infrastructure.
