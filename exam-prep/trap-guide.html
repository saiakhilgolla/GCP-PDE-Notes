Exam Traps

🚨 BigQuery Partition Pruning Trap

Trap: Using DATE(timestamp_column) in WHERE clause disables partition pruning.

Fix: Use DATE(timestamp_column) = '2023-01-01' or timestamp_column >= '2023-01-01'

🚨 External Table Limitations

Trap: Trying to use streaming inserts or DML on external tables.

Reality: External tables are read-only. Use native tables for writes.

🚨 Pub/Sub Message Ordering

Trap: Assuming global message order without ordering keys.

Reality: Only messages with same ordering key are ordered. Different keys = parallel processing.

🚨 Dataflow Autoscaling Confusion

Trap: Setting max workers too low for streaming jobs.

Rule: Streaming jobs need headroom. Set max workers 2-3x expected steady state.

🚨 Cloud Storage Consistency

Trap: Expecting immediate consistency for list operations.

Reality: List operations are eventually consistent. Use object notifications for real-time awareness.

🚨 Bigtable Hot-spotting

Trap: Using sequential row keys (timestamps, incremental IDs).

Fix: Reverse timestamps, use hash prefixes, or field promotion patterns.

🚨 Dataproc Preemptible Economics

Trap: Using 100% preemptible workers for critical jobs.

Best Practice: Mix regular + preemptible. 50% preemptible saves costs while maintaining reliability.

🚨 IAM Role Confusion

Trap: Using primitive roles (Owner, Editor, Viewer) in production.

Principle: Always use predefined or custom roles following least privilege.
