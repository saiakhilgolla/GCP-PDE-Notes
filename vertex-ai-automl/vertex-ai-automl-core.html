1. Core Concepts

What is AutoML?

Automated machine learning service that enables users with limited ML expertise to train high-quality models. AutoML handles feature engineering, architecture search, hyperparameter tuning, and model selection automatically.

Key Principles

  • No-code ML: Train models without writing ML code
  • Neural Architecture Search (NAS): Automatically finds optimal model architectures
  • Transfer Learning: Leverages pre-trained models as starting points
  • AutoML Tables: Handles structured/tabular data with automated feature engineering
Key Insight: AutoML is designed for users who want ML results without deep ML expertise, while Custom Training is for ML engineers who need full control.

4. Exam Traps

Common Gotchas

  • Budget vs Time: Training budget is in node-hours, not wall-clock time. Higher budget = more parallel training.
  • Data Quality: AutoML requires clean data. It won't fix fundamental data quality issues.
  • Feature Types: AutoML Tables auto-detects feature types but can be wrong. Always review and adjust.
  • Imbalanced Data: AutoML handles some imbalance but severe imbalance (>100:1) needs preprocessing.

AutoML vs Custom Training Confusion

Scenario Choose AutoML When Choose Custom Training When
Team Expertise Limited ML expertise ML engineers available
Time to Market Need quick results Can invest in development
Model Architecture Standard architectures work Need custom architecture
Data Size Within AutoML limits Very large datasets (>100GB)
Feature Engineering Automatic FE sufficient Need domain-specific features

Model Evaluation Traps

  • Precision vs Recall: AutoML optimizes for different metrics. Know which metric matters for your use case.
  • Threshold Selection: Default threshold may not be optimal. Review precision-recall curves.
  • Feature Importance: Shows correlation, not causation. Don't assume causal relationships.
  • Overfitting: High training accuracy with low test accuracy indicates overfitting despite AutoML safeguards.
5. Quick Cheatsheet

AutoML Quick Facts

  • Tabular: Max 200M rows, 1000 columns, 100GB. Uses TabNet + ensembles.
  • Images: Min 1000 examples per label. Exports to Edge TPU, Core ML, TF.js.
  • Text: BERT-based models. Classification, sentiment, entity extraction.
  • Video: Action recognition, object tracking. Docker export only.
  • Training Budget: More budget = more architecture search, not longer training.
  • Feature Engineering: Automatic for tabular: scaling, encoding, feature crosses.
