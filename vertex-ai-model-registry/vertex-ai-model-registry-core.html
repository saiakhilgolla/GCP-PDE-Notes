Vertex AI Model Registry

Comprehensive study notes for GCP Professional Data Engineer exam covering model versioning, deployment, governance, and evaluation.

1. Core Concepts

Model Registry Overview
  • Centralized repository for ML models across the ML lifecycle
  • Supports model versioning with semantic versioning (1.0.0)
  • Provides model lineage tracking from training to deployment
  • Integrates with Vertex AI Pipelines, Training, and Endpoints
  • Supports multiple frameworks: TensorFlow, PyTorch, Scikit-learn, XGBoost
Model Versions & Aliases
  • Model versions are immutable snapshots of trained models
  • Aliases are mutable pointers to specific versions (e.g., "champion", "challenger")
  • Default alias is default pointing to latest version
  • Aliases enable blue-green deployments without changing endpoint code
  • Version format: projects/PROJECT/locations/LOCATION/models/MODEL/versions/VERSION
Labels & Metadata
  • Labels are key-value pairs for organization and filtering
  • Support for custom metadata including training metrics
  • Automatic capture of training job metadata when using Vertex AI Training
  • Model cards for documentation and compliance
  • Evaluation metrics stored as metadata for comparison
Model Lineage
  • Tracks data lineage from datasets to trained models
  • Links models to training jobs, datasets, and artifacts
  • Provides audit trail for compliance and debugging
  • Integrates with Vertex ML Metadata for full pipeline tracking
  • Supports impact analysis when datasets change

  • Blue-Green: Use aliases to switch traffic between versions
  • Canary: Split traffic percentage between versions
  • A/B Testing: Compare model performance in production
  • Shadow: Run new version alongside current without serving
  • Champion-Challenger: Continuous evaluation pattern

4. Exam Traps

Common Misconceptions
  • Trap: Model Registry is NOT the same as Artifact Registry
  • Trap: Aliases are mutable pointers, versions are immutable
  • Trap: Model artifacts stored in Cloud Storage, not in Registry directly
  • Trap: Cannot delete a model version if it's deployed to an endpoint
  • Trap: Model lineage requires ML Metadata service to be enabled
BigQuery ML Integration Traps
  • Critical: ML.TRANSFORM applies preprocessing from CREATE MODEL
  • Trap: Export BQML → Cloud Storage → Vertex AI for <100ms serving
  • Trap: TRANSFORM clause prevents training-serving skew
  • Trap: ML.EXPLAIN_PREDICT returns Shapley attributions
  • Trap: BigQuery ML.PREDICT cannot achieve sub-100ms latency
Deployment Gotchas
  • Trap: Need both model AND endpoint for online serving
  • Trap: Batch prediction doesn't require endpoints
  • Trap: Blue-green uses aliases, not traffic
