🎯 Core Concepts

What is Dataplex?

Dataplex is a unified data governance platform that organizes, catalogs, and manages data assets across GCP (and on-premises). It combines:

  • Data organization: Lakes, zones, and assets
  • Data discovery: Catalog and search capabilities
  • Data quality: Automated quality checks and monitoring
  • Data lineage: Track data flow and transformations
  • Governance: Metadata management and access control
📍 Lakes, Zones, and Assets (Hierarchy)

Lake: Top-level container for related data. Think "data silo" or "project boundary".

  • Example: "Finance Lake", "Marketing Lake"
  • One lake per data domain
  • Encompasses all zones within that business area

Zone: Subdivision within a lake that groups data by processing stage or purpose.

  • RAW/SOURCE: Unprocessed, ingested data. Read-only for most users.
  • CURATED/GOVERNED: Cleaned, validated, business-ready data. Owner enforces quality standards.
  • PUBLISH/CONSUMPTION: Final aggregated views for analytics, reporting, ML.

Asset: Individual data resource (dataset, table, file, or schema) within a zone.

  • Example: BigQuery table `projects/myproject/datasets/finance_curated/tables/revenue_daily`
  • Each asset can have attached metadata, schema, quality rules, and lineage
  • Assets are discoverable through Dataplex Catalog
Lake (Finance) ├─ Zone (RAW) │ ├─ Asset: BigQuery table `raw_transactions` │ ├─ Asset: GCS folder `gs://bucket/raw_transactions/` │ └─ Asset: Looker view `raw_transactions_view` ├─ Zone (CURATED) │ ├─ Asset: BigQuery table `transactions_clean` │ └─ Asset: BigQuery table `transactions_validated` └─ Zone (PUBLISH) ├─ Asset: BigQuery table `revenue_daily` └─ Asset: Data Studio dataset `revenue_metrics`
🔍 Data Discovery & Catalog

Dataplex Catalog provides a searchable inventory of all data assets across GCP.

Key Features:

  • Full-text search: Find assets by name, description, tag, owner
  • Automated metadata extraction: Schema, row counts, last modified, location
  • Custom metadata & tags: Add business context (e.g., "PII", "quarterly", "critical")
  • Glossary terms: Define and link business vocabulary to assets
  • Data classification: PII, sensitive, public labels
  • Rich previews: Sample rows, schema, quality metrics in one view

Catalog Entry: A discoverable record for an asset. Contains:

  • Technical metadata (schema, format, location)
  • Business metadata (owner, description, SLA)
  • Quality metrics (null %, cardinality)
  • Lineage (upstream sources, downstream consumers)
✅ Data Quality Framework

Dataplex Quality uses rule-based checks to validate data continuously.

Quality Check Types:

Check Type Example Rule Use Case
NULL_COUNT email_address < 5% nulls Completeness
DUPLICATE_COUNT user_id has no duplicates Uniqueness
FRESHNESS updated_at ≤ 24 hours old Timeliness
SCHEMA_CHANGE Schema hasn't changed Integrity
ROW_COUNT 100K ≤ rows ≤ 1M Volume validation
RANGE price > 0 AND price < 10000 Value bounds
REGEX email matches [a-z]@[domain].com Format validation
STAT_DEVIATION Today's avg value ±10% of 7-day avg Anomaly detection

Quality Score: Percentage of records passing all applicable quality checks. Visible in Dataplex UI and exportable to Dataflow for remediation workflows.

🔗 Data Lineage

Dataplex automatically tracks data flow across pipelines to show upstream sources and downstream consumers.

Lineage Sources:

  • Dataflow/Beam: Auto-captured when pipelines use BigQuery IO connectors
  • BigQuery: SQL queries (INSERT, CREATE AS SELECT) tracked via job metadata
  • Dataform: Explicit lineage from ref() and self() functions
  • Dataproc: Spark/PySpark lineage when configured
  • Manual: Custom lineage via Catalog API
