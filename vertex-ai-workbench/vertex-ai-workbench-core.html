Vertex AI Workbench - Study Notes

1. Core Concepts
Concept Description Key Points
Managed Notebooks Fully serverless Jupyter environment Auto-shutdown on idle, no VM management, BigQuery integration
User-Managed Notebooks Full control over underlying VM SSH access, custom environments, persistent instance
JupyterLab Web-based interactive development environment Notebooks, terminal, file manager, extensions
Idle Shutdown Automatic VM termination when inactive Managed notebooks only, configurable timeout
Executor Managed service for running notebooks as jobs Serverless execution, scheduled runs, parameterization
Key Distinction: Managed = Serverless + Auto-shutdown | User-Managed = Full Control + Persistent

4. Exam Traps
Trap 1: "Auto-shutdown on idle" or "no VM management" → Always choose Managed Notebooks
Trap 2: "Custom GPU drivers" or "SSH access needed" → Always choose User-Managed Notebooks
Trap 3: Don't confuse Workbench with Vertex AI Training - Workbench is for interactive development, not production training
Trap 4: Managed notebooks have a 12-hour continuous runtime limit - for longer jobs, use Executor or User-managed

Common Confusion Points

Confused With Key Difference Decision Signal
Colab Enterprise Workbench = VM-based, Colab = serverless runtime "Persistent storage" → Workbench
Vertex AI Training Workbench = interactive, Training = batch jobs "Jupyter notebook development" → Workbench
Cloud Functions Workbench = long-running, Functions = event-driven "Data science workflow" → Workbench
Dataproc Workbench = single-user, Dataproc = cluster "Individual data scientist" → Workbench

Trick Question Patterns

  • Cost optimization: Managed notebooks auto-shutdown saves costs vs always-on user-managed
  • Security requirements: No SSH = managed, SSH required = user-managed
  • Custom environments: Special libraries/drivers = user-managed
  • Team sharing: Neither type supports multi-user - use separate instances
  • Production workloads: Use Executor for scheduled notebook runs, not interactive sessions
