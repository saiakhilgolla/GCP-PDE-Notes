Vertex AI Matching Engine

Core Concepts

Vector Search Fundamentals

Matching Engine is Google's managed vector similarity search service for finding semantically similar items using machine learning embeddings.

Key Components

  • Embeddings: High-dimensional vectors representing data (text, images, audio)
  • Index: Data structure optimizing vector similarity search
  • Endpoint: Deployed index for serving queries
  • Query: Input vector to find similar items

Vector Similarity Metrics

MetricUse CaseRange
CosineText embeddings, normalized vectors[-1, 1]
Euclidean (L2)Image embeddings, spatial data[0, ∞]
Dot ProductRaw embeddings, recommendation(-∞, ∞)

Search Algorithms

  • ANN (Approximate Nearest Neighbor): Fast, scalable, slight accuracy trade-off
  • Brute Force: Exact results, slower, suitable for small datasets

Exam Traps

Common Gotchas

  • Index Rebuild: Adding/removing dimensions requires full index rebuild
  • Distance Metrics: Cannot change after index creation
  • Regional Limitation: Index and endpoint must be in same region
  • Brute Force Scaling: Only suitable for <10M vectors

Tricky Scenarios

  • Streaming Updates: Only supported on Tree-AH indexes, not brute force
  • Multi-tenancy: Use filtering, not separate indexes per tenant
  • Cold Start: First queries after deployment may have higher latency
  • Dimension Mismatch: Query vector dimensions must exactly match index

Quick Cheatsheet

Index Selection

  • Tree-AH: >10M vectors, high QPS, streaming updates
  • Brute Force: <10M vectors, exact results required

Distance Metrics Quick Guide

  • Cosine: Text embeddings, document similarity
  • L2 (Euclidean): Image embeddings, spatial data
  • Dot Product: Recommendation systems
