1. Core Concepts

What is Cloud Scheduler?

A fully managed cron job scheduler for GCP. Triggers automated tasks on a schedule using cron syntax. Runs once at a specified time or repeatedly on an interval. Integrated with Cloud Pub/Sub, HTTP endpoints, and App Engine targets.

📌 How Cloud Scheduler Works

Execution Flow:

  1. You define a Cloud Scheduler job with a cron expression and a target (HTTP, Pub/Sub, App Engine)
  2. Cloud Scheduler service monitors the time
  3. When scheduled time arrives, Cloud Scheduler invokes the target
  4. Target executes (HTTP request sent, Pub/Sub message published, or App Engine task queued)
  5. Execution tracked in Cloud Logging; failures trigger retries if configured

Key Principle: Cloud Scheduler is serverless and regional. You don't manage infrastructure; you define jobs declaratively. Region determines where the job runs.

📌 Cron Syntax Fundamentals

Standard Unix Cron Format (5 fields):

┌───────────── minute (0-59) │ ┌───────────── hour (0-23) │ │ ┌───────────── day of month (1-31) │ │ │ ┌───────────── month (1-12) │ │ │ │ ┌───────────── day of week (0-6, Sunday=0) │ │ │ │ │ * * * * *

Common Examples:

Expression Meaning
*/5 * * * * Every 5 minutes
0 9 * * * Daily at 9:00 AM (all months)
0 9 * * MON-FRI Weekdays at 9:00 AM
0 2 1 * * First day of month at 2:00 AM
30 10,14,18 * * * 10:30 AM, 2:30 PM, 6:30 PM every day
0 0 * * * Daily at midnight (UTC)

Important: All times are in UTC by default. Cloud Scheduler does NOT have native timezone support — convert to UTC or use application-level logic.

📌 Job Targets: Types & Behavior
Target Type Use Case Invocation
HTTP External APIs, webhooks, Cloud Run services, custom servers HTTP POST (or GET/PUT/DELETE) to a public or internal URL
Pub/Sub Decouple scheduler from workers, fan-out to many subscribers Publishes a message to a Cloud Pub/Sub topic
App Engine Legacy App Engine applications (GAE Standard/Flexible) Routes request to specified App Engine service/version

HTTP Target Example: Schedule a daily report export by triggering a Cloud Run endpoint at 2 AM UTC.

Pub/Sub Target Example: Publish a message to a topic every minute; multiple Cloud Functions or workers subscribe and process independently.

Regional Deployment

Cloud Scheduler is a regional service. Each job is created in a specific region (e.g., us-central1). The scheduler service in that region manages the job. For high availability, create duplicate jobs in multiple regions with identical schedules.
