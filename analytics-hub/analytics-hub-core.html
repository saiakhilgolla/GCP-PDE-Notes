1. Core Concepts

What is Analytics Hub?

Analytics Hub is a data exchange platform that enables secure sharing of BigQuery datasets within and across organizations without copying or moving the underlying data. It provides a marketplace model for data products with fine-grained access control and always-fresh data visibility.

Core Components & Definitions
Component Description Key Characteristics
Data Exchange Container for organizing data listings Private (org/VPC-SC) or public (discoverable by anyone). Published by organizations.
Listing Published dataset or subset within an exchange Points to BigQuery dataset. Includes description, documentation, terms. Created by publisher.
Linked Dataset Read-only view created when subscriber subscribes to listing Created in subscriber's BigQuery project. Always reflects latest data. Zero copy. Zero egress cost.
Publisher Organization sharing BigQuery data Creates exchange. Publishes listings. Controls access. Owns underlying dataset.
Subscriber Organization consuming shared data Discovers and subscribes to listings. Accesses via linked dataset. Read-only access.
No Data Copying — Key Advantage

Why "No Copying" Matters:

  • Always Fresh: Linked dataset always reflects publisher's latest data in real-time.
  • No Storage Cost: Subscriber doesn't pay for duplicate storage.
  • No Egress Cost: Data doesn't leave publisher's project — zero egress fees.
  • No Sync Pipelines: No ETL needed to keep copies in sync.
  • Single Source of Truth: Publisher's data is the only copy.

Commercial Data Exchange

Google's public Analytics Hub marketplace contains commercial datasets published by third-party providers. Examples: financial market data, weather data, demographic data, consumer insights. Subscribers can enrich analytics without building custom ingestion pipelines.

2. Architecture & Components

Data Exchange Architecture Flow

Layer Component Role Organization
Publisher Side BigQuery Dataset Source data (owned by publisher) Publisher Org
Data Exchange + Listing Publicly/privately published data product Publisher Org
Platform Analytics Hub Service Manages discovery, subscription, access control Google Cloud
Subscriber Side Linked Dataset (read-only) Virtual view of publisher's dataset Subscriber Org
BigQuery Queries Query linked dataset like any BQ dataset Subscriber Org

Cross-Organization Data Sharing

How Cross-Org Sharing Works

Scenario: Org A (Publisher) shares with Org B (Subscriber)

  1. Step 1 - Publisher: Org A creates Data Exchange and Listing pointing to BigQuery dataset.
  2. Step 2 - Discovery: Org B discovers listing (if public) or receives invitation (if private).
  3. Step 3 - Subscribe: Org B subscribes to listing via Analytics Hub console/API.
  4. Step 4 - Linked Dataset: Analytics Hub creates a linked dataset in Org B's BigQuery project.
  5. Step 5 - Query: Org B queries the linked dataset. Queries execute against Org A's data.
  6. Step 6 - Revoke: Org A revokes access anytime; Org B loses linked dataset access.
