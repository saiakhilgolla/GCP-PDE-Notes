Core Concepts

What is Explainable AI?

Explainable AI provides insights into ML model predictions by showing which features contributed most to each prediction.

Key Methods

  • Feature Attributions: Assigns importance scores to input features
  • SHAP (SHapley Additive exPlanations): Unified framework for feature importance
  • Integrated Gradients: Path-based attribution method
  • Sampled Shapley: Approximation of Shapley values for efficiency
Core Purpose: Answer "Why did the model make this prediction?" by quantifying feature contributions

Request Flow

  1. Client sends prediction request to endpoint
  2. Model generates prediction
  3. Attribution engine computes feature importance
  4. Response includes prediction + attributions
Explanation Metadata Structure

Explanation metadata defines how to interpret model inputs and outputs:

Input Metadata

  • input_baselines: Reference point for attribution (e.g., zeros, mean)
  • modality: Data type (numeric, categorical, image)
  • encoding: How features are encoded

Output Metadata

  • output_names: Names of prediction outputs to explain
  • index_display_name_mapping: Human-readable class names
Critical: Metadata must match model's input/output schema exactly

Exam Traps

Common Misconceptions
TRAP: Explainable AI does NOT improve model accuracy - it only explains predictions
TRAP: Feature attributions are relative, not absolute importance scores
TRAP: Explanations require the model to be deployed with explanations enabled - can't be added post-deployment
Configuration Gotchas
  • Wrong baseline: Using zeros for all features when mean baseline would be better
  • Missing metadata: Not defining explanation metadata causes deployment failures
  • Method mismatch: Using sampled-shapley for real-time when integrated-gradients is sufficient
  • Scale confusion: Comparing attribution scores across different models or datasets
Key Pattern: If asked about real-time explanations with low latency, choose integrated-gradients over sampled-shapley
