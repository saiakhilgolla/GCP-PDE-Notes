1. Core Concepts

What is Cloud Audit Logs?

Cloud Audit Logs is a fully managed service that maintains logs of actions taken on Google Cloud resources. It answers "who did what, where, and when" across your GCP environment. Audit logs are automatically generated, stored, and queryable without requiring explicit enablement for most services.

Core Purpose: Track API calls, changes to security configurations, access patterns, and system events to ensure compliance, security, and troubleshooting.
Four Types of Audit Logs
Log Type Description Default Retention Enabled By Default
Admin Activity API calls that create, modify, or delete resources. Examples: creating VMs, updating IAM policies, deleting databases. 400 days ✓ Always
Data Access API calls that read data or metadata. Examples: listing storage buckets, reading Firestore documents, querying BigQuery tables. 30 days ✗ Must enable
System Event Google-initiated actions not in response to user requests. Examples: GCP scaling actions, maintenance, automatic backups. 400 days ✓ Always
Policy Denied API calls denied by IAM policy or VPC Service Controls. Captures access attempts that failed due to policy enforcement. 400 days ✗ Must enable
Log Entry Structure

Each audit log entry contains:

  • timestamp — when the action occurred (UTC)
  • protoPayload — the audit action details (API method, resource, requestor, status)
  • resourceName — the GCP resource affected
  • severity — ERROR, WARNING, INFO, DEBUG, DEFAULT
  • sourceIP — IP address of the caller
  • userEmail — authenticated user identity
  • serviceName — the GCP service (compute.googleapis.com, storage.googleapis.com, etc.)
  • methodName — the API method invoked
  • request — the API request payload (if stored)
  • response — the API response payload (if stored)
  • status — success or failure code
Key Fields for Filtering: Use timestamp, protoPayload.methodName, protoPayload.authenticationInfo.principalEmail, and severity in queries.
Admin Activity vs Data Access Logs
Aspect Admin Activity Data Access
What's Logged Write operations: create, update, delete resources Read operations: list, get data
Cost Free (included) Billable ($0.50 per GB ingested)
Volume Lower volume, infrequent Very high volume (every read generates logs)
Use Cases Compliance, change tracking, security audits Data access audits, regulatory requirements (HIPAA, PCI-DSS)
Enable Status Always enabled Explicitly enable per service

2. Architecture & Components

Audit Log Flow Architecture
GCP API Call → Audit Logging Service → Cloud Logging Storage → Log Router → Multiple Destinations

1. User/service makes an API call to a GCP resource
2. GCP API intercepts the call and generates an audit log entry
3. Entry is written to Cloud Logging backend (Cloud Audit Logs bucket in _Default organization)
4. Log Router applies inclusion/exclusion filters
5. Logs route to sinks (BigQuery, Cloud Storage, Pub/Sub, Splunk, etc.)
6. Organization-level logs viewable in Cloud Console and gcloud CLI

Log Sinks

A sink is a configuration that routes logs from Cloud Logging to external destinations.

Destination Type Use Case Key Benefits
BigQuery Dataset Long-term analysis, complex queries, dashboards Queryable, integrates with BI tools, cost-effective for large volumes
Cloud Storage Bucket Archival, regulatory compliance, retention policies Low cost, immutable object lock, lifecycle management
Pub/Sub Topic Real-time alerts, streaming processing, automation Triggers Cloud Functions, fan-out to multiple subscribers
Logging Bucket (Cloud Logging) Extended retention, custom bucket configs Separate retention from _Default, organized by environment
Splunk, Datadog, etc. Third-party SIEM integration Unified monitoring across multi-cloud
Sink Filter Language: Sinks use the Cloud Logging Query Language (similar to LogQL). Example filter: resource.type="gce_instance" AND severity="ERROR"
Log Buckets

Log buckets are Cloud Logging-native storage for logs with configurable retention and querying.

Bucket Type Default Location Retention Use Case
_Default Global (multi-region) 30 days (customizable) All logs by default
_Required Global 400 days (fixed) Admin Activity, System Event, Policy Denied logs
Custom Regional or multi-region 1 day to 3650 days Org-specific retention requirements
_Required Bucket: Automatically stores Admin Activity, System Event, and Policy Denied logs for 400 days. Cannot be disabled or deleted. This is a compliance mandate.
