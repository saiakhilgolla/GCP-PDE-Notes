Vertex AI Custom Training

Core Concepts

Training Jobs

Custom Training Job: A managed service to train ML models using your own training code. Supports TensorFlow, PyTorch, scikit-learn, XGBoost, and custom frameworks.

Job Type Use Case Key Features
Custom Training Full control over training logic Custom containers, distributed training, hyperparameter tuning
AutoML Automated model development No code required, built-in feature engineering

Container Types

Pre-built Containers: Google-managed containers with popular ML frameworks (TensorFlow, PyTorch, scikit-learn) pre-installed. Fastest to get started.
Custom Containers: Your own Docker container with specific dependencies, frameworks, or custom code. Pushed to Artifact Registry or Container Registry.

Training Pipeline

1. Data: Input data from GCS, BigQuery, or other sources

2. Container: Pre-built or custom container with training code

3. Compute: VMs with CPUs, GPUs, or TPUs

4. Output: Trained model artifacts saved to GCS

Exam Traps
Trap: "Use AutoML for all custom training needs"
Reality: AutoML is for automated model development. Custom Training is for when you need full control over training logic, custom architectures, or specific frameworks.
Trap: "Pre-built containers can't be customized"
Reality: You can install additional packages in pre-built containers using requirements.txt or setup.py, but for complex dependencies, use custom containers.
Trap: "Hyperparameter tuning always improves model performance"
Reality: Poor search space definition or insufficient trials can lead to suboptimal results. Define reasonable bounds and run enough trials.
Trap: "More parallel trials = faster hyperparameter tuning"
Reality: Bayesian optimization needs sequential feedback. Too many parallel trials reduce optimization effectiveness.
Trap: "TPUs work with all frameworks"
Reality: TPUs are optimized for TensorFlow. PyTorch support exists but is limited. Use GPUs for PyTorch workloads.

Common Confusion Points

Often Confused Key Difference
Custom Training vs AutoML Custom = your code, AutoML = automated
Pre-built vs Custom containers Pre-built = Google managed, Custom = you build
GPU vs TPU GPU = versatile, TPU = TensorFlow optimized
Distributed vs Hyperparameter tuning Distributed = parallel compute, HP tuning = parameter optimization

Key Commands

  • gcloud ai custom-jobs create - Submit training job
  • gcloud ai hp-tuning-jobs create - Start hyperparameter tuning
  • gcloud ai jobs list - List training jobs
  • gcloud ai jobs cancel - Cancel running job

Best Practices

  • Use pre-built containers for standard frameworks
  • Implement checkpointing for long training jobs
  • Start with single-node before distributed training
  • Monitor resource utilization and costs
  • Use appropriate machine types for workload
