1. Core Concepts

What is Transfer Appliance?

Transfer Appliance (TA) is a physical hardware device designed to transfer massive amounts of data to Google Cloud Storage offline. It bridges the gap when network bandwidth is insufficient or prohibitively expensive for large-scale data migrations.

  • Offline transfer: Ship the device; no network dependency
  • High-capacity: TA40 (40TB) and TA480 (480TB) models available
  • Automated data capture: Built-in encryption, compression, and deduplication
  • Enterprise-grade: Secure enclosure, tamper-evident seals, audit logs
  • Cost-effective: One device shipment cheaper than 100s of Mbps internet for weeks
How Transfer Appliance Works (End-to-End)
  1. Order: Create Transfer Appliance job in Cloud Console, specify destination Cloud Storage bucket
  2. Receive: Device arrives at your data center with pre-loaded encryption keys
  3. Connect: Plug into network (Ethernet; requires IP, gateway, DNS config)
  4. Mount & Transfer: Mount NFS or copy via command-line; device captures data in real-time
  5. Finalize & Ship: Verify data checksum; seal device; ship back to Google
  6. Ingest: Google receives device, verifies integrity, uploads to Cloud Storage bucket
  7. Verify: Compare source and destination checksums; job completes

Transfer Appliance Architecture Layers
Hardware Layer
  • Storage: SSD/HDD arrays with RAID protection
  • Encryption: Hardware security module (HSM) with AES-256
  • Physical Security: Tamper-evident seals, intrusion detection
Network Layer
  • NFS Server: Mount via NFS 4.0/4.1 for file access
  • Data Transfer: No direct internet; local network only
  • Management Interface: Web UI + command-line tool
Data Capture Layer
  • Ingestion: Real-time data capture during copy/mount
  • Compression: Optional gzip/bzip2 (configurable)
  • Deduplication: Automatic for identical blocks
  • Checksums: MD5/SHA-256 per file
Software/Control Layer
  • Job Tracking: Cloud Console shows status, logs
  • Audit Logging: All operations logged for compliance
  • Verification: Source-destination checksum validation
