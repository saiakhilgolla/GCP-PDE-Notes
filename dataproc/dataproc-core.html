GCP Dataproc Study Notes

Core Concepts

What is Dataproc?

Managed Apache Spark and Hadoop service on GCP. Fully managed clusters that auto-provision VMs, install software, and handle scaling.

Key Processing Frameworks

  • Apache Spark: In-memory distributed processing for batch and streaming
  • Apache Hadoop: MapReduce and HDFS for large-scale batch processing
  • Apache Hive: SQL-like queries over distributed datasets
  • Pig, Presto, Zeppelin: Additional data processing tools

Cluster Modes

Mode Use Case Duration Characteristics
Standard Long-running clusters Days/Months Persistent storage, reusable
Single Node Development/Testing Short No HDFS replication, cheaper
Preemptible Batch workloads Variable 80% cost savings, fault-tolerant

Serverless Spark

Fully managed Spark without cluster management. Auto-scales from 0. Pay only for resources used during job execution.

Network Architecture

  • Internal IPs for node communication
  • Optional external IPs for SSH access
  • Private IP clusters supported
  • VPC firewall rules auto-created

Exam Traps

Common Misconceptions

Trap: Thinking Dataproc is serverless by default

Reality: Standard Dataproc requires cluster management. Only Serverless Spark is truly serverless.

Trap: Using 100% preemptible workers

Reality: Need at least 2 primary workers for HDFS replication and cluster stability.

Trap: Expecting instant autoscaling

Reality: Scaling takes 2-5 minutes due to VM provisioning time.

Frequently Confused Concepts

Concept A Concept B Key Difference
Dataproc Dataflow Dataproc = managed clusters, Dataflow = serverless streaming
Primary workers Preemptible workers Primary = guaranteed, Preemptible = can be terminated
Workflow template Job Template = reusable definition, Job = single execution
Cluster mode Serverless Cluster = manage VMs, Serverless = no infrastructure

Pricing Gotchas

  • Charged for full minutes even if job runs seconds
  • Preemptible instances billed minimum 1 minute
  • Storage charges separate from compute
  • Network egress charges apply
  • Idle clusters still incur compute charges
Quick Cheatsheet

Decision Framework

Use Dataproc when: Migrating existing Spark/Hadoop, need cluster control, custom configurations

Use Serverless Spark when: New projects, simple Spark jobs, want zero infrastructure management

Use BigQuery when: SQL analytics, no complex ETL, structured data
