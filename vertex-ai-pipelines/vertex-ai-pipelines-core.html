Vertex AI Pipelines - GCP PDE Study Notes

1. Core Concepts

What is Vertex AI Pipelines

Vertex AI Pipelines is a serverless ML workflow orchestration service built on Kubeflow Pipelines (KFP). It allows you to automate, monitor, and govern ML systems by orchestrating workflows in a serverless manner.

  • Serverless execution - no infrastructure management
  • Kubeflow Pipelines compatibility
  • TensorFlow Extended (TFX) integration
  • Built-in experiment tracking and metadata management
  • Pipeline versioning and scheduling
Kubeflow Pipelines (KFP)

Open-source platform for building and deploying portable, scalable ML workflows based on Docker containers.

  • Components - containerized functions that perform specific tasks
  • Pipelines - workflows composed of connected components
  • Pipeline runs - single execution of a pipeline
  • Experiments - collections of pipeline runs
TensorFlow Extended (TFX)

Production ML platform providing components and libraries for deploying production ML pipelines.

Components and Artifacts

Components are self-contained pieces of code packaged as Docker images. They consume and produce artifacts.

Component Types:

  • Lightweight components - Python functions with @component decorator
  • Containerized components - Full Docker containers
  • TFX components - Pre-built TensorFlow Extended components

Artifact Types:

  • Dataset - Training/evaluation data
  • Model - Trained ML models
  • Metrics - Evaluation metrics
  • Examples - TFX data format
  • Schema - Data schema definitions
Pipeline Caching

Vertex AI Pipelines automatically caches component outputs to avoid re-running unchanged components.

  • Automatic caching based on component inputs and code
  • Cache key includes component spec, inputs, and container image digest
  • Can be disabled per component or pipeline
  • Speeds up iterative development and re-runs
