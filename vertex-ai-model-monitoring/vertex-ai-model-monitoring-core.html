Skew Detection

Training-Serving Skew: Statistical differences between training dataset and production serving data

Types of Skew

  • Schema Skew: Different feature schemas between training and serving
  • Distribution Skew: Statistical distribution differences in feature values
  • Feature Skew: New features in serving not present in training
Detection Methods:
  • L-infinity distance for numerical features
  • Jensen-Shannon divergence for categorical features
  • Chebyshev distance for distribution comparison
Drift Detection

Data Drift: Changes in input feature distributions over time in production

Drift TypeDescriptionImpact
Covariate ShiftP(X) changes, P(Y|X) stays sameModel accuracy degrades
Prior Probability ShiftP(Y) changes, P(X|Y) stays samePrediction distribution shifts
Concept DriftP(Y|X) changesModel becomes obsolete
Baseline: Reference dataset used for comparison. Can be:
  • Training dataset
  • Previous time window of serving data
  • Custom validation dataset

Exam Traps

Common Gotchas

  • Skew vs Drift: Skew compares training vs serving; Drift compares serving over time
  • Baseline Selection: Training data for skew detection, previous serving data for drift
  • Attribution Overhead: Feature attribution significantly increases prediction latency
  • Categorical Features: Require different distance metrics than numerical features
  • Sample Rate Impact: Lower rates reduce accuracy but improve performance

Quick Cheatsheet

Setup Requirements

  • Model deployed to Vertex AI Endpoint
  • Request-response logging enabled
  • Baseline dataset in BigQuery/GCS
  • Feature schema definition

Distance Metrics

  • L-infinity: Max absolute difference (numerical)
  • Jensen-Shannon: Distribution divergence (categorical)
  • Chebyshev: Maximum feature deviation

Alert Conditions

  • Distance metric > threshold
  • Missing feature detection
  • Schema validation failures
  • Attribution score anomalies

Best Practices

  • Monitor top 10-20 most important features
  • Set conservative thresholds initially
  • Use training data as skew baseline
  • Enable request-response logging
