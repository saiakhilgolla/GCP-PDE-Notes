Cloud Storage - Study Notes

1. Core Concepts

What is Cloud Storage?

Unified object storage service for developers and enterprises. Store any amount of data and retrieve it as often as needed.

Key Components

  • Buckets: Containers that hold your data (objects)
  • Objects: Individual pieces of data stored in buckets
  • Projects: Organize and control access to buckets
  • Storage Classes: Different price/performance tiers

Storage Classes

Class Use Case Min Duration Availability Retrieval Cost
Standard Frequently accessed data None >99.95% Free
Nearline Monthly access 30 days 99.95% $0.01/GB
Coldline Quarterly access 90 days 99.95% $0.02/GB
Archive Annual access/backup 365 days 99.95% $0.05/GB

Location Types

  • Region: Single geographic region (lowest latency)
  • Dual-region: Two specific regions (99.95% availability)
  • Multi-region: Large geographic area (highest availability)

4. Advanced Features

Object Lifecycle Management

  • Age: Days since object creation
  • CreatedBefore: Specific date threshold
  • IsLive: Current vs. noncurrent versions
  • MatchesStorageClass: Current storage class
  • NumberOfNewerVersions: Version count threshold

Object Versioning

  • Each object write creates new generation number
  • Live object is latest version
  • Noncurrent versions retained until deleted
  • Can restore previous versions
  • Delete markers for deleted objects

Customer Managed Encryption Keys (CMEK)

  • Use Cloud KMS keys for encryption
  • Set at bucket or object level
  • Automatic key rotation support
  • Regional keys must match bucket region
  • Cannot change after object creation

Transfer Options

Method Use Case Speed

Transfer Appliance Large datasets, offline Physical shipping
Parallel Composite Uploads Large files Faster than single stream
5. Exam Traps

Storage Class Traps

  • TRAP: Nearline has 30-day minimum - early deletion charges apply
  • TRAP: Archive retrieval costs $0.05/GB - expensive for frequent access
  • TRAP: Regional vs Multi-regional pricing differs significantly
  • TRAP: Lifecycle transitions are one-way only (Standard→Archive)
