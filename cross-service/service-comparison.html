Exam Traps - Common Confusion Points

BigQuery vs Cloud SQL

TRAP: Using BigQuery for OLTP or Cloud SQL for analytics

  • BigQuery: Analytics, OLAP, read-heavy, columnar, serverless
  • Cloud SQL: OLTP, transactional, write-heavy, row-based, managed instance
  • Key decision: Workload pattern (analytical vs transactional)

Dataflow vs Dataproc

TRAP: Choosing based on familiarity instead of requirements

  • Dataflow: Serverless, Apache Beam, streaming-first, auto-scaling
  • Dataproc: Managed VMs, existing Spark/Hadoop code, batch-heavy
  • Key decision: Serverless vs managed cluster preference

Storage Class Confusion

TRAP: Mixing up BigQuery external tables with Cloud Storage

  • Native BigQuery: Best performance, full features, higher cost
  • External Tables: Data stays in Cloud Storage, lower performance
  • BigLake: External with security features, S3
