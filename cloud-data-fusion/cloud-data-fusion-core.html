1. Core Concepts

What is Cloud Data Fusion?

Cloud Data Fusion is a fully managed, cloud-native data integration service built on top of CDAP (Cask Data Application Platform). It provides a visual, code-free interface for building ETL/ELT pipelines at scale.

Key Idea: Combines visual designer, pre-built plugins, and enterprise-grade data integration capabilities to move, transform, and integrate data across sources.
CDAP Foundation

Cloud Data Fusion runs on CDAP, an open-source framework that provides:

  • Pipeline Execution Engine: Runs batch and real-time pipelines at scale
  • Plugin System: Extensible architecture with 200+ pre-built connectors and transformations
  • Data Pipeline Studio: Visual drag-and-drop interface for pipeline design
  • Metadata Management: Tracks data lineage, schema, and transformations
  • REST API: Programmatic pipeline deployment and orchestration
Pipeline Types
Pipeline Type Execution Model Use Case Engine
Batch Scheduled or triggered execution Daily ETL, periodic data loads, bulk transformations Spark or MapReduce
Real-time Continuous event streaming Live data feeds, message queue processing, event-driven architectures Spark Streaming or Beam
Replication Incremental change data capture Keep data warehouses in sync, replicate database changes CDC (Change Data Capture)
Key Components: Sources, Transforms, Sinks

Sources (Plugins)

  • Read data from external systems
  • Examples: BigQuery, Cloud SQL, GCS, Pub/Sub, HTTP, Salesforce, SAP, Oracle
  • Support batch and streaming modes

Transforms (Plugins)

  • Process, enrich, and reshape data
  • Examples: Aggregator, Joiner, Filter, Deduplicator, SQL transform, Python script
  • Chainable in directed acyclic graph (DAG) patterns

Sinks (Plugins)

  • Write transformed data to destinations
  • Examples: BigQuery, Cloud SQL, GCS, Pub/Sub, Data Catalog
  • Auto-create tables or append to existing
Visual Designer

Cloud Data Fusion Studio provides a drag-and-drop interface:

  • Canvas: Design pipelines by connecting plugins visually
  • Plugin Library: Browse 200+ pre-built sources, transforms, and sinks
  • Configuration Panels: Configure credentials, schemas, transformations without code
  • Data Preview: Test data flow at each stage before running the full pipeline
  • Lineage View: Visualize end-to-end data flow and dependencies
  • Schema Management: Auto-detect or manually define input/output schemas
Plugins Deep Dive

Plugin Types

  • Source Plugins: BigQuery, Cloud SQL, PostgreSQL, MySQL, Oracle, Salesforce, SAP HANA, Cloud Storage (CSV, Parquet, Avro), Pub/Sub, Kafka, HTTP
  • Transform Plugins: SQL transform (write SQL queries), Python script (custom transformations), Aggregator, Joiner (INNER/LEFT/RIGHT/FULL), Filter (row filtering), Deduplicator, Validator, Column Renamer, Type Converter
  • Sink Plugins: BigQuery (auto table creation, schema matching), Cloud SQL, Cloud Storage (output format options), Pub/Sub, Data Catalog, Bigtable

Custom Plugins

  • Write Java or Scala code to create custom source, transform, or sink plugins
  • Package as JAR and deploy to Cloud Data Fusion
  • Integrated into pipeline canvas like built-in plugins
Transformations & Data Processing

Common Transformation Patterns

  • SQL Transform: Write standard SQL for complex transformations, filtering, aggregations
  • Python Script: Row-level transformations in Python, access schema and transform each row
  • Joiner: INNER, LEFT, RIGHT, FULL OUTER joins on configured keys
  • Aggregator: GROUP BY operations, SUM, AVG, COUNT, MIN, MAX
  • Filter: Include/exclude rows based on column conditions
  • Deduplicator: Remove duplicate rows on specified columns
  • Conditional Router: Route records to different outputs based on conditions

Schema Handling

  • Auto-detect from source or manually define
  • Types: boolean, int, long, float, double, string, bytes, date, datetime, decimal
  • Nullable columns supported
Replication & Change Data Capture

Replication keeps a target system in sync with source changes in near real-time or incremental batches.

Replication Types

  • Full Load: Initial complete copy of source to target
  • Incremental (CDC): Track and replicate only changed rows via change data capture mechanisms
  • Snapshot: Time-point copies at scheduled intervals

CDC Support

  • MySQL, PostgreSQL, Oracle, SQL Server using binary logs or WAL (Write-Ahead Logs)
  • BigQuery (export snapshots or use time-based increments)
  • Tracks INSERT, UPDATE, DELETE operations
Replication Pipeline Workflow:
1. Configure source database connection
2. Select tables to replicate
3. Choose full load + incremental or incremental-only strategy
4. Configure CDC method (binary logs, WAL, transaction log)
5. Set target connection and table creation behavior
6. Schedule or run continuous replication
