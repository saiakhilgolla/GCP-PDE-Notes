Core Concepts

What is BigQuery ML?

BigQuery ML enables data scientists and analysts to build and run ML models directly in BigQuery using SQL. No need to move data or learn Python/R.

Supported Model Types

Model Type Use Case Notes
LINEAR_REG Predict continuous numeric values Sales, revenue, prices
LOGISTIC_REG Binary or multi-class classification Churn yes/no, category prediction
KMEANS Unsupervised clustering No label column needed
ARIMA_PLUS Time series forecasting Automatic seasonality detection
RANDOM_FOREST_* Tree-based models CLASSIFIER or REGRESSOR variants
BOOSTED_TREE_* XGBoost-style gradient boosting Often highest accuracy on tabular data
MATRIX_FACTORIZATION Collaborative filtering Recommendation systems
AUTOML_* Vertex AI AutoML via SQL Best of both worlds
TENSORFLOW Import pre-trained models From Cloud Storage

Key ML Functions

  • ML.EVALUATE: Returns evaluation metrics (AUC-ROC, precision, recall for classifiers; MAE, MSE, RMSE, R² for regressors)
  • ML.PREDICT: Run inference on new data with predicted labels and probabilities
  • ML.EXPLAIN_PREDICT: Returns Shapley feature attributions with predictions
  • ML.TRANSFORM: KEY Applies preprocessing transformations to new data
  • ML.FEATURE_INFO: Feature statistics including distribution and null rate
  • ML.FORECAST: ARIMA_PLUS future predictions with confidence intervals
  • ML.WEIGHTS: Inspect learned coefficients for linear models
Architecture

BigQuery ML Architecture Components

Component Function Location
Model Training Executed within BigQuery compute BigQuery slots
Model Storage Stored as BigQuery objects Within dataset
Feature Engineering TRANSFORM clause preprocessing Saved with model
Prediction Serving ML.PREDICT function calls BigQuery query engine
Model Export TensorFlow SavedModel format Cloud Storage → Vertex AI

Feature Engineering Pipeline

TRANSFORM Function Purpose Example
ML.STANDARD_SCALER Normalize to mean=0, stddev=1 Continuous features
ML.MIN_MAX_SCALER Scale to range [0,1] Bounded normalization
ML.ONE_HOT_ENCODER Categorical to binary vectors String categories
ML.BUCKETIZE Numeric to categorical buckets Age groups, price tiers
ML.FEATURE_CROSS Interaction features Category1 × Category2

BQML to Vertex AI Export Workflow

  1. Train: CREATE MODEL in BigQuery ML using SQL
  2. Export: Export model to Cloud Storage as TensorFlow SavedModel
  3. Import: Import SavedModel into Vertex AI Model Registry
  4. Deploy: Deploy to Vertex AI Endpoint for sub-100ms online predictions
Use this workflow when you need BigQuery ML training but require online serving latency < 100ms (which BQML cannot achieve)
