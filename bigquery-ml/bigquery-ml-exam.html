BigQuery ML - GCP PDE Exam Prep

Core Concepts

BigQuery ML enables SQL-based machine learning directly in BigQuery. Train models using SQL, no Python/R required. Models stored as BigQuery objects.

Key Workflow

  1. CREATE MODEL - Train model with TRANSFORM clause for preprocessing
  2. ML.EVALUATE - Check model performance metrics
  3. ML.PREDICT - Run inference on new data
  4. ML.TRANSFORM - Apply same preprocessing to serving data

🚨 Exam Traps

Model Types Confusion

  • BOOSTED_TREE_* typically highest accuracy on tabular data, NOT RANDOM_FOREST_*
  • MATRIX_FACTORIZATION for recommendations, NOT collaborative filtering in name
  • ARIMA_PLUS for time series, NOT just ARIMA
  • AUTOML_* uses Vertex AI behind scenes, NOT native BQML algorithm

Function Purpose Traps

  • ML.TRANSFORM applies TRAINING preprocessing to NEW data - prevents skew
  • ML.EXPLAIN_PREDICT for feature attribution, NOT just predictions
  • ML.FORECAST only for ARIMA_PLUS models, NOT general predictions
  • ML.WEIGHTS only for linear/logistic, NOT tree models

BQML vs Vertex AI Decision

  • Use BQML when: Data in BigQuery, batch predictions OK, SQL-preferred workflow
  • Use Vertex AI when: Need <100ms latency, complex preprocessing, non-SQL users
  • Export workflow: Train in BQML → Export to GCS → Import to Vertex AI → Deploy endpoint

Feature Engineering Gotchas

  • TRANSFORM clause must be in CREATE MODEL, NOT in ML.PREDICT
  • ML.STANDARD_SCALER = mean 0, stddev 1 (NOT min-max scaling)
  • ML.MIN_MAX_SCALER = range 0 to 1 (NOT standardization)
  • Feature crosses only between categorical columns

Essential SQL Patterns

-- Create model with preprocessing
CREATE MODEL `project.dataset.model_name`
OPTIONS(model_type='BOOSTED_TREE_CLASSIFIER')
AS SELECT * FROM ML.TRANSFORM(
  SELECT ML.STANDARD_SCALER(feature1) OVER() as feature1,
         ML.ONE_HOT_ENCODER(category) OVER() as category,
         label
  FROM `training_table`
);

-- Batch prediction
SELECT * FROM ML.PREDICT(
  MODEL `project.dataset.model_name`,
  TABLE `prediction_data`
);

-- Evaluation metrics
SELECT * FROM ML.EVALUATE(
  MODEL `project.dataset.model_name`
);

Transform Functions

Function Purpose Output Range
ML.STANDARD_SCALER Normalize to mean=0, std=1 (-∞, +∞)
ML.MIN_MAX_SCALER Scale to range [0, 1]
ML.ONE_HOT_ENCODER Categorical to binary {0, 1}
ML.BUCKETIZE Numeric to categorical Bucket indices
ML.FEATURE_CROSS Categorical interactions Combined features
