GCP Professional Data Engineer: Cloud Data Catalog

1. Core Concepts

Cloud Data Catalog is GCP's fully managed, scalable metadata repository. It discovers, catalogs, and governs data across BigQuery, Cloud Storage, Pub/Sub, Dataproc, and on-premises data sources. It bridges technical metadata (schemas, lineage) with business metadata (business glossary, data stewardship).

Metadata Types & Organization

Technical Metadata: Data type, schema, table size, creation date, column names, data source location

Business Metadata: Data owner, steward, purpose, audience, classification level, glossary terms

Operational Metadata: Last modified, access patterns, query frequency, lineage dependencies

Tags & Tag Templates

Tag Template is a reusable schema for tags. Define once, apply many times. Templates have mandatory and optional fields.

Tags are instances of a template attached to assets (tables, columns, datasets). Tags add business context and enforce data governance policies.

Example: Create template pii_classification with fields: level (HIGH/MEDIUM/LOW), owner, retention_days. Apply this tag to sensitive columns across multiple tables.

Template Scope: Templates are project-level resources. Assets can be tagged with templates from their own project or inherited from parent organizations if using centralized metadata management.

Policy Tags & Column-Level Security

Policy Tags are hierarchical tags that control data access at the column level in BigQuery. Unlike business tags, policy tags enforce actual access control.

How it works:

  • Create a taxonomy (hierarchical structure of policy tags)
  • Assign policy tags to BigQuery columns
  • Grant roles/datacatalog.categoryEditor or roles/datacatalog.categoryMasker to users/groups
  • Users with masking role see NULL or redacted data; editors see actual values

Key difference from IAM: Policy tags are column-specific. IAM dataset permissions are coarse-grained. Use policy tags for fine-grained access within datasets.

Data Lineage & Dependency Tracking

Lineage shows upstream sources and downstream consumers of data assets. Automatically captured for BigQuery tables created via queries.

Lineage sources:

  • BigQuery query logs (table read/write)
  • Dataflow job definitions
  • Dataproc Spark/Hive jobs
  • Custom lineage via API (Data Catalog Lineage API)

Common use case: Track impact of schema changes. If column is dropped, Data Catalog shows all dependent dashboards and reports.

Search & Discovery

Unified Search queries across all metadata—table names, column names, descriptions, tags, glossary terms, lineage.

Search syntax: name:"customer" tag:pii=HIGH

Discovery patterns:

  • Find tables by business owner (tag-based search)
  • Locate PII columns across organization
  • Identify data dependencies before pipeline changes
  • Search by glossary term to find canonical data definitions

2. Architecture & Components

Cloud Data Catalog operates as a centralized metadata service. It does not move or store actual data—only metadata and governance policies.

Key Components

Component Purpose Key Role
Entry A data asset (BigQuery table, Cloud Storage bucket, Pub/Sub topic) Basic unit of cataloging
Entry Group Logical container for related entries Organize entries by source or domain
Tag Template Schema defining tag structure and fields Enforce consistent metadata
Tag Instance of template applied to entry Attach business context
Policy Tag Taxonomy Hierarchical classification system Enforce column-level access control
Glossary Term Business definition (e.g., "customer lifetime value") Standardize business language
Lineage Edge Dependency link between assets Track data flow

Integration Architecture

GCP Service Auto-Discovery Lineage Support Policy Tag Support
BigQuery ✓ Automatic ✓ Query-based ✓ Full support
Cloud Storage ✓ Via connector ✓ Custom lineage API ✗ No column-level
Pub/Sub ✓ Automatic ✓ Manual registration ✗ No
Dataflow ✗ Manual ✓ Via API ✗ No
On-Premises ✗ Manual + connectors ✓ Via API ✗ No

Permission Model

Role Permissions Use Case
roles/datacatalog.viewer Read entries, tags, search metadata Data consumers, analysts
roles/datacatalog.editor Create/edit entries, tags, glossary Data stewards, catalogers
roles/datacatalog.admin Full control + policy tag taxonomies Data governance leads
roles/datacatalog.categoryEditor Edit policy tags, view unmasked data Authorized data users
roles/datacatalog.categoryMasker Apply masking, view masked data Data privacy admins
