🎯 Core Concepts

Apache Beam & Dataflow

  • Apache Beam: Unified programming model for batch and streaming
  • Dataflow: Fully managed Apache Beam runner on GCP
  • Portability: Beam code runs on multiple runners (Dataflow, Spark, Flink)
  • SDK Support: Java, Python, Go

Windowing & Time

  • Fixed Windows: Non-overlapping time intervals
  • Sliding Windows: Overlapping time intervals
  • Session Windows: Activity-based grouping with gaps
  • Global Window: Single window containing all elements

Watermarks & Triggers

  • Watermark: Progress indicator for event time
  • Late Data: Data arriving after watermark
  • Triggers: When to emit window results

⚠️ Exam Traps

🚨 Common Gotchas

  • Templates vs Flex Templates: Classic templates are JSON, Flex are Docker containers
  • Streaming Autoscaling: Less aggressive than batch, CPU-based by default
  • Late Data: Requires allowed lateness AND triggers to handle properly
  • Side Inputs: Must fit in worker memory, broadcasted to all workers
  • Stateful Processing: Only works with keyed data (after GroupByKey)

Windowing Traps

  • Default Window: Global window with default trigger (end of window)
  • Session Window Gap: Minimum 1 minute gap required
  • Sliding Window Size: Must be multiple of period
  • Late Data in Batch: All data is "on time" - no late data concept

Template Confusion

  • Classic Templates: Staging location, JSON metadata, limited flexibility
  • Flex Templates: Docker image, runtime parameters, full flexibility
  • When to Use Each: Flex for complex logic, Classic for simple pipelines

Performance Traps

  • Hot Keys: Uneven key distribution causes stragglers
  • Fusion: Automatic optimization can hide performance issues
  • Streaming Insert Mode: Use FILE_LOADS for BigQuery streaming at scale
  • Worker Types: Streaming needs more memory (n1-standard-2 default)

Window Type Code Example Use Case

Common Transforms

  • Map: 1:1 element transformation
  • FlatMap: 1:N element transformation
  • Filter: Conditional element selection
  • GroupByKey: Shuffle operation, creates iterables
  • CoGroupByKey: Join multiple PCollections
  • Combine: Reduce operation (sum, max, custom)
