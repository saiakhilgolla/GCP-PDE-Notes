🔄 Storage Transfer Service (STS)

Complete PDE Exam Study Guide

What is Storage Transfer Service?

A managed service for transferring data to Google Cloud Storage from external sources (on-premises, other clouds, URL lists). Built for large-scale, one-time or recurring transfers with scheduling, filtering, and job monitoring.

1. Core Concepts

What STS Does

  • Managed data transfer — automates large-scale data movement to GCS
  • Multiple sources — on-premises, AWS S3, Azure Blob, HTTP/HTTPS URLs, GCS
  • Scheduling — one-time or recurring transfers on custom schedules
  • Filtering — include/exclude patterns, date ranges, metadata filters
  • Manifest file — list of specific objects to transfer instead of directory scan
  • On-premises agent — required for non-cloud sources without public URLs
Transfer Job Lifecycle
  1. Create — define source, destination, schedule, filters
  2. Enable — activate the job (disabled by default)
  3. Execute — STS schedules and runs transfers
  4. Monitor — track progress, errors, retries in Cloud Logging
  5. Complete — automatic on final run or manual deletion
Source Types & Requirements
Source Requirements Agent Needed?
AWS S3 Access keys, bucket permissions No
Azure Blob Connection string, container access No
On-premises NAS/Server SMB, NFS, or local filesystem Yes
HTTP/HTTPS URLs Manifest file with URL list No
GCS → GCS Source bucket permissions No
On-Premises Agent (Conn)

What is it? A containerized or VM-based agent deployed in your on-premises data center. Acts as a bridge between local storage and GCS.

  • Docker or GKE compatible — runs as container
  • Connects to local storage — NFS, SMB, local filesystem
  • Uses service account — to authenticate with GCS
  • Network security — private network, no public IP required
  • HA setup — multiple agents for redundancy

When to use: Transferring large amounts of on-premises data that lacks public internet access. Can operate on private networks with VPN or Direct Connect.

Manifest Files

What is it? A CSV or newline-delimited JSON file listing specific objects to transfer instead of scanning entire directory.

Format (CSV):

source_path,source_metadata_updated_time
/data/file1.txt,2024-01-15T10:30:00Z

Benefits:

  • Transfers only specified files, not entire bucket
  • Faster than full directory scan for large datasets
  • Allows selective retry of failed objects
  • Useful for incremental transfers with known object list
Transfer Filters & Patterns

Available filters:

  • Include patterns — *.jpg, logs/**/*.txt (glob syntax)
  • Exclude patterns — temp/*, .tmp
  • Modified time range — only objects modified after/before date
  • Size range — min/max object size in bytes
  • GCS versioning — how to handle versioned objects

Note: Filters are evaluated during transfer; if 100GB matches filter, all 100GB is transferred.

Job Execution Model
  • Asynchronous — jobs run independently, managed by STS backend
  • Scheduled execution — based on cron expression or one-time
  • Parallel transfers — multiple files transferred concurrently
  • Retry logic — automatic retries for transient failures
  • Checkpointing — resumable transfers if interrupted
