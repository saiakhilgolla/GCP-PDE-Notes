Dataproc Workflow Templates

1. Core Concepts

What are Workflow Templates?

Define a DAG of Dataproc operations (create cluster, submit jobs, delete cluster) as a single managed unit. Lighter-weight than Cloud Composer for simple Dataproc-only pipelines.

Key Components

Component Purpose Details
Template Definition Blueprint for workflow Cluster config + job sequence + parameters
Instantiation Execute the template Via gcloud, REST API, or Cloud Scheduler
Managed Cluster Ephemeral compute Created, used, deleted automatically
Job Sequence Ordered operations Spark, Hadoop, Hive, Pig jobs

When to Use vs Other Services

Scenario Best Service Reason
Single recurring SQL BigQuery Scheduled Query Native, serverless, cost-effective
Single Dataflow job on schedule Cloud Scheduler + Dataflow template Simple scheduling without Airflow overhead
Simple Dataproc-only workflow Dataproc Workflow Templates Linear DAG, no cross-service dependencies
Multi-service complex pipeline Cloud Composer Complex dependencies, branching logic
2. Architecture

Template Structure

Layer Components Function
Template Definition YAML/JSON spec Defines cluster config, job sequence, parameters
Cluster Management Managed Cluster or Cluster Selector Auto-create ephemeral or use existing cluster
Job Execution Ordered job list Sequential execution of Spark/Hadoop jobs
Parameter Injection Runtime parameters Template parameterization for reusability

Execution Flow

Step Action Component
1 Template instantiation triggered gcloud/API/Scheduler
2 Cluster creation (if managed) Dataproc service
3 Job submission and execution Sequential job processing
4 Cluster deletion (if managed) Automatic cleanup

Cluster Options

Option Use Case Lifecycle
Managed Cluster Ephemeral workflows Created → Used → Deleted automatically
Cluster Selector Persistent clusters Use existing cluster by labels

4. Exam Traps

Common Mistakes

  • Trap: Using Composer for simple Dataproc-only pipelines
  • Reality: Workflow Templates are lighter-weight for Dataproc-only workflows
  • Trap: Assuming Workflow Templates can orchestrate multiple services
  • Reality: Only supports Dataproc jobs - use Composer for cross-service workflows
  • Trap: Expecting complex branching logic
  • Reality: Linear job execution only - no conditional branching
  • Trap: Confusing with Dataflow templates
  • Reality: Different services - Dataproc vs Dataflow job orchestration

Decision Framework Traps

If You See... Wrong Answer Correct Answer
"Simple recurring Spark job" Cloud Composer Dataproc Workflow Templates
"ETL from GCS to BigQuery with transformations" Workflow Templates Dataflow or Composer (multi-service)
"Complex dependencies between services" Workflow Templates Cloud Composer
"Conditional job execution based on data" Workflow Templates Cloud Composer with Airflow operators

Cost Trap

Warning: Composer has minimum cost (GKE + Cloud SQL) even when idle. Don't over-engineer simple workflows that could use Workflow Templates or Cloud Scheduler.
